贰 字符串、向量和数组

发表于 2017-08-27 | 更新于 2017-08-28 | 分类于 C++ primer | 阅读次数:
| 字数统计: 769

书在这一章主要讲解了string和vector的使用，对于替换char *和type *的决心还是很明显的。。。毕竟标准库的实现相对来说确实方便与安全了很多，但是高校目前还没有普及吧，我记得学校开设的C++课程里就从来没有提到过这个。（也可能真的是我们学校太low了）

下面来看，string对象会自动忽略开头所有形式的空白，空格符、换行符、制表符都算，然后会从第一个真正的字符开始读起，到下一处空白处停止。所以如果你真的想打印空格的话需要用到getline操作，它只会在遇到换行符的时候才结束读操作。

而且string类在做加法的时候有一点需要注意，当你把string对象和字符字面值以及字符串字面值混在一条语句中使用时，必须确保每个加法运算符的两侧的运算对象至少有一个是string，至于为什么，到后面就会知道啦。

书中在本章提到了一个个人认为非常好用的工具，我也在这安利一波。为了进行更方便的列表对象遍历，可以使用C++11提供的语句：范围for语句。语句的语法形式为：

for( declaration: expression)
    statement
for(auto b: expression) //常规的拷贝取值
for(auto &c: expression) //引用取值，用于进行变量的修改

此处的declaration部分负责定义一个变量，因为是在已知列表对象中进行遍历，所以用auto就会显得很方便。在不修改元素值的情况下单纯的拷贝即可，如果涉及到列表元素值的修改则需要进行引用，如auto&。

vector部分暂时先只做提及，其相关部分操作按照书上的内容来使用就差不多了。对于替换常规的int []来说，vector真的是非常方便的。除了不能直接通过下标形式添加元素，因为只能使用push_back。

本书在练习3.26提了这样一个问题：

在100页的二分搜索程序中，为什么用的是mid = beg + (end - beg) / 2,而非mid = (beg + end) / 2;?

最初的时候我看到这个问题脑海中浮现的画面是在坐标轴上，两个坐标相减可以表明两者之间的长度，而相加其实是没有什么意义的。这个时候还沾沾自喜觉得自己不愧是学物理的，后来发现真正的原因是迭代器支持的运算（P.99)根本就不包括加法运算，也就是说迭代器只支持迭代器之间的减法运算，而不支持两个迭代器相加这种运算。

C风格的数组和字符串暂时先不写了，原因是个人单方面不是很喜欢这种方式。后期应该会进行update，所以此处未完待补。
