壹 变量和基本类型

发表于 2017-08-25 | 更新于 2017-08-26 | 分类于 C++ primer | 阅读次数:
| 字数统计: 2,059

首先需要了解到的一点是，算术类型的尺寸在不同的机器上是有所差别的，显而易见的就是在32位机器和64位机器上long的尺寸是不一样的，32位机器是32位长度而64位机器的long是64位长度的。所以在考虑到移植性情况下的代码应该尽量规范使用类似于char16_t, int32_t等的类型说明。

当我们选取定义一个变量时，对其类型的选择有以下一些小建议：

  1. 进行计数等变量的定义时，选择无符号类型，因为你已知这些变量不可能为负数值。同时在变量名称的声明的时候也力求规范，比如使用u、_前缀进行限定说明（u_var)。
  2. 对整型进行选择时，如果范围不是很大（小于int的上下限时，上限：2147483647 为2^31 -1，下限即-2^31）时应该使用int类型，而一旦通过幂次判断变量会大于这个值时直接选择long long类型，原因是在32位机器中，long和int的尺寸相同并没有提升，而在64位机器中，long和long long的尺寸相同，所以为了方便大家都使用long long就可以了。
  3. 除了在逻辑运算中使用char和bool之外，算术运算应该避免使用char和bool类型，因为不同机器对于char的类型判定是不同的，有些机器是有符号类型而有些是无符号类型。这种不明确的行为非常容易产生令人意想不到的错误。
  4. 当小数出场的时候，毫不犹豫选择double类型吧，因为目前来看双精度的计算代价较之单精度的而言十分微小，甚至有些场合略有胜出，因此double类型是定义小数类型的不二选择。

默认的浮点型字面值是一个double。

对于字符变量定义时，单引号'A'和双引号"A"是有区别的，单引号的对象表示一个char类型，而双引号"A"表示的实际上是字符构成的array，同时，这个array的内置长度并不是1，字符串会在末尾加上结尾符'\0'，所以实际这个字符串的长度为2。

转义序列就不多说了，常见的\n, \t等，有意思的是我们可以使用泛化的表示形式来表达转义序列，比如通过十六进制或者八进制进行转义说明，此时\x后接数字是十六进制的表示形式，而\后接1-3个数字是八进制的表示形式。

当你想显式指定字面值的类型时，要注意前缀和后缀的使用对象和意义是不相同的，前缀用于char类型，且大小写敏感，u和U分别表示的是Unicode16位和32位字符，而后缀用来匹配整型和浮点型字面值，其大小写是不敏感的，但实际上普遍使用大写形式，因为小写的l实在是太容易混淆1而引发错误。

初始化和赋值的差别是什么？个人的观点是，初始化实际上包含了两个步骤，先是在内存空间创建这个对象，然后在这个地址上写入初始化的值。赋值操作是不相同的，因为赋值是找到这个已存在的对象，然后擦去这个对象地址先前有的值之后写入新的值。

一个小的注意点就是在对变量进行初始化时可以进行列表初始化，即用花括号表示数值，值得注意的就是在列表初始化时如果有数据丢失的数据类型转换，则此时列表初始化会进行报错，int val = 3.14是可行的，而int val = {3.14} 是不可行的！

extern是特意用来声明变量的哦，但是一旦存在赋值操作，则会抵消extern的作用，即extern int val = 100，也是定义而不仅仅是声明了。变量可以重复声明但是不可以重复定义。

浅谈一下命名规范的问题。说实话，首先英语功底得扎实，真的会有想起变量名但是不知道英文该怎么说的时候，比如女朋友会有juzhen这个变量名然后抛弃matrix(我这么说她应该不会打我吧，逃），然后第一个考虑的是，是否是用于宏定义的命名，如果是宏定义的命名的话那么一般情况下都是尽量大写的，多用宏定义可以加强代码规范啊同时在修改代码的时候也会提供一些便利。其次一般的变量命名就用小写就可以了，除了不能使用关键字之外请不要使用大写开头或者全都是大写字母，虽然原则上没有问题但是在可读性方面会产生不利影响。有一些特殊的情况，比如在用户自定义的类命名的时候，使用大写字母开头的形式Sales_item啊， Bulk_quote啊之类的，所以这就是为什么普通变量命名不要用大写开头，会和类的命名冲突啊。多个单词之间一定要有一种机制进行隔断，要么是用下划线_，要么后续单词的字母开头大写比如salesData这种，个人倾向于下划线因为还是要更清楚些。最后直接引用《C++ primer》上的原话：

对于指针和引用，二者都是用来提供间接的访问，而定义指针时有个小建议就是令其与引用一样初始化时对其进行值定义，如果就是一个暂时留着以备后续使用的指针，也可以用nullptr进行初始化，这样是为了防止后续产生的问题。关于引用和指针在面对常量时的操作其实很容易记忆，一个const的引用和指针，那么他引用或者指向的对象是无所谓是否为常量的，而非常量引用和指针则不能针对常量对象。而在考虑到顶层const和底层const时，首先要认识到的一点是，底层的const是和复合类型如引用与指针相关的，如果对于单一类型的常量，则不管如何他都是顶层const，而对于复合类型来说，顶层const意味着该指针与引用本身无法改变，而底层是指他们所指向的对象不能改变。

auto用得好的话，会发现定义变量类型突然就变得友好起来了。但是，人家也不是老好人，他也有他的规矩：

  1. auto对于限定多个变量的类型时，必须确保他们的类型都是一致的。
  2. auto会自动忽略顶层const限制，这个特性在对于复合类型对象的定义时十分好理解，但是如果面对单一类型变量的时候就不是那么友好了：

如果想偷懒直接使用表达式或者函数类型来定义变量的类型，又不想实际计算表达式或者运行函数的时候，可以用decltype，爽是爽了，但是它的处理方式和auto是不同的。牢记一点：decltype会将所指向对象的所有特性都保留下来！记住这点其实也就没了。还有一个就是decltype((variable))的结果永远是引用，而decltype(variable)只有在对象的确是引用的时候才是引用，这也是十分需要注意的地方。
