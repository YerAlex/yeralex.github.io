我回来啦

发表于 2018-05-07 | 更新于 2018-05-07 | 分类于 Nonsense | 阅读次数:
字数统计: 372

我胡汉三又回来啦，不过却已时隔多日。回首一瞧，嘿，都9个月没有更新自己的博客了。

我的脑海里一直回想着一句话：

有些事你可以放一放，但是不能弃掉它。

今日可能也不知怎么就想起来自己还是要有个地儿能记记自己都干了些啥，都想干些啥，免得以后啊，路走的多了走的远了，就忘了自己最初定的方向，偏离了自己的目标。这是一方面，其次的话，我啊毕竟不是个天才，记忆力着实有限，自己的博客就像是日记本，搜罗起来容易而且呐还方便写，毕竟电脑不离身呀，虽然缺了那么点书卷气倒是委实有点可惜。

今儿个我也先不多发表感想，先把最要紧的事做了，你说最要紧的事是啥，嘿嘿，先记下怎么在新电脑上恢复博客啊，这9个月不玩，都忘的个精光啦。

咱也就不琢磨啥个性化啊高端技巧啊之类的了，一步步来吧。总结一下就是下面几个步骤：

  1. 安装git
  2. 生成密钥绑定自己的github和coding
  3. 安装node、npm和hexo
  4. 然后就回归到正常操作啦

我这是不是说的也太简洁啊-，-啊呀没关系，这玩意说下去真的丢人，就这么多吧！

本章一开始在基本概念里谈及了一些后面将会涉及到的内容，例如：重载运算符，左值和右值。重载运算符由于目前还没有谈及类的构造所以暂时不予考虑，而左右值其实简单的区分还是很好分辨的，左值就是一个实体对象，而右值相对于就是该对象地址所写入的值。以上是我个人的观点，大佬如Scott Meyers判别左右值时给出了如下的观点：

A useful heuristic to determine whether an expression is an lvalue is to ask if you can take its address. If you can, it typically is. If you can’t, it’s usually an rvalue. A nice feature of this heuristic is that it helps you remember that the type of an expression is independent of whether the expression is an lvalue or an rvalue.
